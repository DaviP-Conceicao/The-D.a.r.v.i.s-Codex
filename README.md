O **The DARVIS Codex** não é apenas uma documentação técnica; é um ecossistema de aprendizado tratado como código (**Knowledge-as-Code**). O objetivo deste projeto é centralizar comandos, laboratórios e estratégias de infraestrutura, sendo gerenciado e organizado de forma híbrida por mim e pelo meu agente de IA em desenvolvimento, o **D.A.R.V.I.S.**.

---

## 💥 O Diferencial: The Living Documentation
Diferente de repositórios estáticos, o Codex utiliza um motor de ingestão em Python para padronizar novos conhecimentos. 

### 🟢 System Status
*   **Core Engine:** Operacional (Scripts de ingestão locais).
*   **Agent Status:** Em desenvolvimento (Integração de longa duração em progresso).
*   **Knowledge Base:** Ativa e categorizada.

---

## 📂 Estrutura do Repositório

### 🐧 01. Linux & Operações
Focado em gerenciamento de processos, permissões e sobrevivência em terminal.
*   **Destaque:** Guia de Hardening e códigos de estado de processos (STAT).

### 🌐 02. Networking
Fundamentos de conectividade, protocolos TCP/IP e mapeamento de portas críticas.
*   **Destaque:** Diagnóstico de rede e ferramentas de transferência rápida.

### ☁️ 03. Cloud Computing
Modelos de serviço (IaaS, PaaS, SaaS) e responsabilidade compartilhada na nuvem.
*   **Destaque:** Simulação de infraestrutura local com Docker e Python Servers.

### 🔐 04. Security
Práticas de SSH Hardening, auditoria de usuários e firewalls.
*   **Destaque:** Comandos de auditoria (`last`, `lastlog`, `id`) para detecção de acessos não autorizados.

---

## 🤖 Automação & IA (The DARVIS Motor)
O projeto conta com um motor de automação (`core/scripts/codex_auto.py`) que permite:
1.  **Ingestão Padronizada:** Criação de arquivos Markdown com metadados automáticos.
2.  **Organização por Tags:** Classificação automática em pastas específicas.
3.  **Preparação para o Agente:** A estrutura foi desenhada para que o agente D.A.R.V.I.S. possa ler e escrever via API no futuro.
